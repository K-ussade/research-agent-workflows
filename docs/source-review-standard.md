# Source Review Standard

## Purpose

This standard defines how sources are reviewed before they support a public research brief or publication.

## Source Review Fields

| Field | Requirement |
| --- | --- |
| Source title | Name of the source. |
| Source URL or citation | Public URL or formal citation when available. |
| Source type | Primary, secondary, official, academic, news, dataset, documentation, or other. |
| Access status | Public, paywalled, private, sealed, or unavailable. |
| Copyright handling | Link and paraphrase unless quotation is short, necessary, and reviewed. |
| Relevance | What claim or context the source supports. |
| Reliability note | Why the source is appropriate or limited. |
| Reviewer | Human reviewer or review role. |
| Approved use | Approved, limited, rejected, or needs review. |

## Source Rules

- Prefer primary sources when precision matters.
- Do not include scraped copyrighted content.
- Do not paste paywalled articles or private documents into this repository.
- Do not cite private Foundation operations, donor records, student records, customer material, or sealed IP in public work.
- Mark uncertain sources as `Draft / review-required`.

## Copyright-Safe Handling

Use short excerpts only when necessary and permitted by review. Prefer paraphrase, source links, and citation metadata.
