# Status Language

## Purpose

Status labels prevent research workflow documents from implying publication, release, evidence, or review that has not happened.

## Status Labels

| Status | Meaning |
| --- | --- |
| Planned | Intended or designed, not yet created or published. |
| Scaffolded | Structure exists, but content may be incomplete or review-required. |
| Scaffolded / published | Public repository exists with initial standards, templates, examples, and validation. |
| Synthetic | Example uses fictional, placeholder, or fabricated-safe content. |
| Draft / review-required | Work is not approved for publication or external claim use. |
| Source-reviewed | Sources have been reviewed for the stated use. |
| Editorial-approved | Human editorial reviewer approved the artifact for the stated use. |
| Published | Public artifact exists and is linked. |
| Paused | Work is intentionally stopped pending review or priority. |
| Deprecated | Artifact remains visible with archival or replacement context. |
| Private / Not public | Material is intentionally excluded from public surfaces. |

## Claim Rules

- Use `Synthetic` for examples.
- Use `Draft / review-required` until human editorial review is complete.
- Use `Published` only when a public artifact exists.
- Do not use AI-generated text or validation success as proof of source accuracy, publication quality, client outcome, or active service delivery.
